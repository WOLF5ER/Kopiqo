# Kopiqo + Supabase: настройка и проверка

## 1. Настройка в Supabase (один раз)

1. **SQL.** Dashboard → SQL Editor → вставьте содержимое `supabase-setup.sql` → Run.
   Скрипт создаст таблицу `finance_data`, включит Row Level Security, создаст
   политики и функцию `nickname_exists`.
2. **Отключите подтверждение email.** Authentication → Sign In / Providers → Email →
   выключите **Confirm email**. Это обязательно: адреса вида `nickname@kopiqo.app`
   служебные, письма на них не доходят, и без этого шага вход после регистрации
   будет невозможен.
3. (Рекомендуется) Authentication → Settings → минимальная длина пароля — оставьте 6.

## 2. Переменные окружения

Проект без сборщика, поэтому роль `.env` играет файл `env.js`:

```js
window.__KOPIQO_ENV__ = {
  SUPABASE_URL: "https://<project-ref>.supabase.co",
  SUPABASE_ANON_KEY: "<publishable/anon ключ>",
};
```

Шаблон — `env.example.js`. Значения уже заполнены для вашего проекта.
Ключ publishable (anon) рассчитан на работу в браузере; данные защищает RLS.

## 3. Запуск

Приложение использует ES-модули — открывать нужно по http(s), не через `file://`:

```bash
npx serve .        # или python3 -m http.server 8000
```

## 4. Как протестировать

**Регистрация**
1. Откройте приложение — появится экран Kopiqo с вкладками Вход / Регистрация.
2. Вкладка «Регистрация», никнейм `wolf5er`, PIN `123456` → «Создать аккаунт».
3. Вы должны автоматически попасть в приложение (без подтверждений и email).
4. Повторная регистрация с тем же никнеймом → ошибка «Этот никнейм уже занят».

**Вход**
1. Нажмите круглую кнопку выхода (слева внизу) → «Выйти».
2. На экране входа введите тот же никнейм и PIN → «Войти» → приложение открылось,
   все данные на месте.
3. Неверный PIN → ошибка «Неверный никнейм или PIN-код».

**Автологин**
1. Войдите, затем полностью закройте вкладку/браузер.
2. Откройте приложение снова — экран входа не показывается, вы сразу внутри.

**Сохранение данных**
1. Добавьте транзакцию/счёт/цель.
2. В Supabase: Table Editor → `finance_data` — в строке вашего пользователя
   поле `data` обновилось (и `updated_at` сдвинулся) в течение ~1 секунды.
3. Откройте приложение в другом браузере (или инкогнито), войдите тем же
   никнеймом и PIN — увидите те же данные.

**Изоляция данных (RLS)**
1. Зарегистрируйте второго пользователя с другим никнеймом.
2. У него будет пустое приложение — чужие данные недоступны ни на чтение,
   ни на запись (политики `auth.uid() = user_id`).

## 5. Примечания

- Данные старой офлайн-версии Kopiqo (localStorage) автоматически переносятся
  в облако при первой регистрации на этом устройстве.
- Внутриигровой PIN-замок приложения (в настройках профиля) — отдельная
  локальная блокировка экрана и никак не связан с PIN-кодом входа.
- Синхронизация отложенная (~0.6 с после изменения) + дослав при закрытии
  вкладки; при обрыве сети изменения хранятся локально и уходят в облако при
  восстановлении соединения. При работе с двух устройств одновременно действует
  правило «последняя запись побеждает».
