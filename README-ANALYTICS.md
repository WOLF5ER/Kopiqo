# Kopiqo Аналитика — страница analytics.html

Отдельная страница анализа и прогноза внутри той же сборки. Ничего в данных
не изменяет — только читает.

## Как открыть
Внутри приложения: круглая кнопка с графиком слева внизу (рядом с кнопкой
«Выйти») открывает аналитику оверлеем поверх Kopiqo — без перехода на другую
страницу; «← в Kopiqo» в шапке закрывает её. Программно: `window.__kopiqoAnalytics()`.

Отдельной страницей: откройте `analytics.html` рядом с основным приложением
(например, `https://ваш-домен/analytics.html`). Используются та же сессия
Supabase и то же облачное хранилище (`storage-sync.js`): если вы уже вошли
в Kopiqo, вход повторно не потребуется, данные подхватятся автоматически
из ключей `finance:state` и `finance:profile` (валюта — из профиля).
Кнопка «← в Kopiqo» в шапке возвращает в приложение.

Если операций в хранилище ещё нет, страница предложит вставить JSON вручную
или посмотреть демо-данные.

## Что внутри
- Обзор: баланс, прогноз расходов месяца с коридором ~80%, безопасный
  дневной лимит, норма сбережений, денежный поток по месяцам, категории,
  заметно отклонившиеся от вашей обычной медианы, нетипично крупные траты.
- Категории: структура месяца против лимитов, профиль трат по дням недели,
  автоматически распознанные регулярные платежи.
- Прогноз: лента расходов до конца месяца, риски по бюджетам, прогноз по
  каждой категории (Хольт с затухающим трендом + темп месяца), баланс на
  6 месяцев вперёд, сроки достижения целей; описание методики — внизу вкладки.

## Технические детали
- Переводы между счетами (`transferId`) и маркеры `account_deleted`
  исключаются из оборотов.
- Лимиты поддерживаются в обоих форматах: `{категория: лимит}` и
  `{счёт: {категория: лимит}}` (суммируются по счетам).
- Пользовательские категории берутся из `customCategories` с их цветами.
- Файлы страницы добавлены в APP_SHELL service worker (кэш v3) — аналитика
  работает офлайн после первой загрузки.
- Исходник модуля: `analytics.src.tsx` не хранится в сборке; страница
  использует скомпилированный `analytics.compiled.js` (тот же import map
  с react/recharts, что и у `app.compiled.js`).
